# 工作区索引

## 全局红线

- 不要无差别读取整个工作区。
- 工作区只保存状态、结论、决策、路径和可复用方法。
- 图片、视频、音频、PDF、表格、代码依赖、缓存、大批量输出物放到外部工作区。
- 不要把 key、token、密码、私钥、服务器 IP、完整 SSH 命令写进工作区。
- 不确定归属、是否删除、是否覆盖时，先问人。

## 按需读取

| 需要 | 读取 |
|---|---|
| 执行具体项目 | 进入项目目录，先读 `_项目规则.md`，再读 `_next.md` |
| 判断项目现状 | 项目的 `_当前状态.md` |
| 查项目背景 | 项目的 `_总览.md` |
| 查重大取舍 | 项目的 `决策日志.md` |
| 整理今日动作 | `40_Daily/_行动池.md` 和当天 Daily |
| 临时输入无归属 | `00_Inbox/` |

## 写回规则

- 下一步变化写项目 `_next.md`。
- 状态变化写项目 `_当前状态.md`。
- 重大转向写项目 `决策日志.md`。
- 可复用经验写 `20_Insights/`。
- 跨项目动作、等待事项、将来也许写 `40_Daily/_行动池.md`。
- 当天要做的行政、生活、临时事务写当天 Daily。

## AI 分工建议

- 高成本模型：最终判断、复杂写作、代码修改、结构设计、发布前检查。
- 低成本模型：整理、提取、初筛、格式化、批量归档。
- 人：目标判断、取舍确认、真实交付、对外承诺。
