# 项目规则

## 本项目红线

- 不读取无关项目文件。
- 不创建、删除、移动无关文件。
- 不把密钥、token、密码、私钥、服务器 IP、客户隐私、完整登录命令写进项目。
- 大文件、导出物、缓存、依赖和批量输出放到外部工作区。
- 不确定是否覆盖、删除、公开时，先问人。

## 项目目标

一句话说明这个项目最终要达成什么。

示例：

> 帮一个真实业务场景接入 AI，让工作从靠人脑硬扛，变成 AI 能参与、能记住、能推进、能复盘的系统。

## 按需读取

| 要做什么 | 读什么 |
|---|---|
| 继续当前动作 | `_next.md` |
| 判断现状 | `_当前状态.md` |
| 查背景和长期目标 | `_总览.md` |
| 查重大取舍 | `决策日志.md` |
| 查方法和模板 | `system/` 或 `templates/` |
| 查单次交付记录 | `work/` 或 `episodes/` |

## 写回

- 下一步变化：`_next.md`
- 状态变化：`_当前状态.md`
- 重大转向：`决策日志.md`
- 可复用方法：`system/` 或上一级知识库
- 临时材料：外部工作区，不写进项目目录

## 默认工作方式

1. 先读规则和当前下一步。
2. 判断任务是否属于本项目。
3. 执行时只改必要文件。
4. 完成后写回结果和新的唯一下一步。
5. 如果产生可复用方法，单独沉淀，不混在流水账里。
