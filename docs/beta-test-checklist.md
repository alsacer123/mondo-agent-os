# 第一轮内测 Checklist

这轮内测只验证一件事：

> 普通用户能不能在不理解技术架构的情况下，把混乱工作口述成一个 AI 能接手的工作现场。

不验证高级自动化，不验证完整场景包，不验证复杂多 Agent 调度。

## 内测前准备

测试者需要准备：

- Cherry Studio 或其他支持 MCP 的 Agent 客户端。
- 一个空的本地工作区路径，例如 `D:/MondoWorkOS`。
- 10-20 分钟完整口述时间。
- 至少 2 个真实项目，不要用虚构项目。

测试者不需要知道：

- MCP 是什么。
- CLI 怎么用。
- AGENTS.md 是什么。
- Daily / 行动池 / `_next.md` 的文件结构。

## 任务 1：开始建立系统

用户在客户端里说：

```text
请开始帮我建立个人 AI 工作系统，工作区放在 D:/MondoWorkOS。
```

预期体验：

- 助手不要先解释技术概念。
- 助手不要让用户先复制模板。
- 助手应该开始问长期方向、当前项目和 AI 边界。

观察点：

- 用户是否知道接下来该说什么。
- 问题是否太像表格，还是像一次自然访谈。
- 用户是否愿意继续讲。

## 任务 2：长期方向口述

用户口述：

- 未来 3-6 个月最重要的方向。
- 当前最混乱、最想让 AI 接住的工作。
- 哪些事情 AI 可以帮忙。
- 哪些事情不能自动做。

预期体验：

- 助手能复述用户的长期方向。
- 助手能识别 AI 边界。
- 助手不要急着创建一堆任务。

成功标准：

- 用户觉得“它听懂了我现在想往哪里走”。

## 任务 3：当前项目口述

用户口述所有真实项目：

- 正在推进的项目。
- 等待中的项目。
- 暂停但仍重要的项目。
- 每个项目现在做到哪一步。
- 每个项目最明确的下一步。

预期体验：

- 助手能区分主线、保活、等待和暂停。
- 助手能把项目事实和下一步分开。
- 助手不要把所有想法都变成今日任务。

成功标准：

- 用户觉得“我手上的事情终于被摊开了”。

## 任务 4：确认初始化草稿

助手生成初始化草稿后，用户检查：

- 长期方向是否像自己。
- 项目有没有漏掉。
- 项目有没有分错。
- 本周主线是否合理。
- 第一个下一步是否能直接执行。

预期体验：

- 助手先让用户确认，再写入。
- 用户能看懂系统准备写什么。
- 用户能指出不准确的地方。

成功标准：

- 用户愿意确认写入工作区。

## 任务 5：写入工作区

用户确认后，助手写入工作区。

预期产物：

- `_Identity/长期方向.md`
- `_Identity/定位.md`
- `_Identity/内容原则.md`
- `40_Daily/_行动池.md`
- 当天 Daily
- 至少一个初始化项目文件夹
- 项目 `_项目规则.md`
- 项目 `_总览.md`
- 项目 `_当前状态.md`
- 项目 `_next.md`
- 项目 `决策日志.md`

成功标准：

- 用户不用理解每个文件，也能知道“我的工作现场已经建起来了”。

## 任务 6：第二天早间整理

用户说：

```text
帮我做今天早间整理。
```

预期体验：

- 助手读取已有工作现场。
- 助手不要求用户重新解释昨天的背景。
- 助手从行动池和项目下一步里抽今天能执行的动作。

成功标准：

- 用户感觉今天不是从空白开始。

## 任务 7：白天新增事项

用户说一条真实新增事项，例如：

```text
我刚和客户聊完，他下周要看方案。
```

预期体验：

- 助手先判断归属。
- 助手说明建议写入哪里。
- 助手不要直接乱写。
- 用户确认后再写入。

成功标准：

- 用户感觉“我随口说的事没有丢，也没有被乱塞进今日任务”。

## 任务 8：晚间收口

用户说：

```text
帮我晚间收口。
```

预期体验：

- 助手询问今天完成了什么。
- 助手判断未完成项应该回流行动池还是明天继续。
- 助手提醒项目状态和决策日志候选。
- 助手生成明天第一步。

成功标准：

- 用户感觉“今天的事被归位了”。

## 记录问题

每个内测用户记录：

- 安装和配置耗时。
- 卡在哪一步。
- 哪个问题听不懂。
- 哪个输出最有帮助。
- 哪个写入建议不准确。
- 用户是否愿意第二天继续用。

## 通过标准

第一轮内测通过，不要求系统完美，只要求：

- 用户能完成初始化建模。
- 用户能看懂初始化草稿。
- 用户能通过确认写入生成工作现场。
- 用户第二天能从早间整理继续。
- 用户能说出 Daily、行动池、项目下一步的大致区别。

如果用户只觉得“这是一个任务记录工具”，则内测失败。

用户必须感受到：

> AI 开始接住我的工作现场，而不是只帮我记待办。
