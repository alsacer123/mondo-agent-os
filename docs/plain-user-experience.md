# 普通用户体验目标

Mondo Agent OS 不应该要求用户理解 Agent 架构、API、RAG、工作流编排或命令行细节。

用户真正需要感受到的是：

1. 我说一句话，系统知道大概该放哪里。
2. 我今天要做什么，不需要每天重新整理。
3. 我做完一件事，下一步能尽快浮出来。
4. 我换一个普通 AI 对话工具，也能读到我的工作现场。
5. 我不需要一次搭完全部系统，先从一个项目跑起来。

## 当前已经补上的体验

- 一键生成最小工作区。
- 一键检查系统是否缺关键文件。
- 一句输入可以初步分流到 Daily、行动池、项目、沉淀区或 Inbox。
- `.mondo/live-state.json` 会把今日 Daily、行动池、项目下一步和当前状态汇总给外部工具。
- `.mondo/agent-context.md` 会把普通 Agent 需要读的上下文变成一份简化说明。

## 仍然没有解决的体验

- 还不能像手机 App 一样开箱即用。
- 还没有图形界面。
- 还没有自动根据用户对话持续改写下一步。
- 还没有适配豆包、DeepSeek 网页版这类普通用户常用入口。
- 还没有把角色包真正做成可安装模块。

## 产品判断

技术能力会继续变化，Codex、DeepSeek、豆包、Claude、OpenHands、Dify 都可能吞掉一部分能力。

但普通用户长期需要的不是“最强 Agent”，而是：

- 一个低门槛入口；
- 一套能记住工作现场的底座；
- 一个会把混乱输入变成下一步的系统；
- 一种可以随着自己工作不断微调的使用方式。

所以 Mondo Agent OS 要优先补的是用户可感知的运行感，而不是堆技术名词。
